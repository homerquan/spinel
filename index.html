Why another JS framework

One reason is author's interest. This framework is trying to adopt up-to-date components and practices and provide a core, light-weighted, and vender neutral tool to make creating web app easier. Another reason is for demo some technologies and practices in the book Webcraft (welcome to like its facebook page

What's spinel

  • It's based on backbone (no more, no less, extendable core)
  • It provides optimized resource management (on-demand loading to every bytes, no block as many as possible)
  • It uses web components for large scale web apps (modulated, easy to test, reusable)
  • It's fast (Pure client-side routing, rendering)

Extra

  • It comes with API server hippie or Spinel mockup API server
